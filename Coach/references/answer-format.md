# Answer Format

Use this structure for final responses.

## 1. Direct answer

- Give the shortest correct answer first.
- Avoid generic filler.

## 2. Evidence

List concrete sources used for claims:

- File paths with line references when available
- Commit hashes and commit subject lines when history is relevant
- Commands used for non-obvious findings

## 3. Explanation

- Connect the evidence to the conclusion.
- Separate facts from inference.

## 4. Onboarding path

- Suggest the next 2-5 files or commands to read/run.
- Order from entrypoint to deeper internals.

## 5. Unknowns and assumptions

- State what could not be verified.
- Ask follow-up questions only when required for correctness.

## Citation requirements

- Every substantive claim should map to at least one cited source.
- If no direct evidence exists, label as inference.
